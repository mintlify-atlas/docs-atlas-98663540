This guide walks you through installing `dd-trace` and capturing your first trace from an Express application.

## Prerequisites

- Node.js >= 18
- A running [Datadog Agent](https://docs.datadoghq.com/agent/) (or set `DD_TRACE_AGENT_URL` to point to your agent)

<Steps>
  <Step title="Install dd-trace">
    Add `dd-trace` to your project using your preferred package manager:

    <CodeGroup>
      ```bash npm
      npm install dd-trace
      ```
      ```bash yarn
      yarn add dd-trace
      ```
      ```bash pnpm
      pnpm add dd-trace
      ```
    </CodeGroup>
  </Step>

  <Step title="Initialize the tracer">
    The tracer **must** be initialized before any other modules are imported. The simplest approach is the `--require` flag — no code changes needed:

    ```bash
    node --require dd-trace/init server.js
    ```

    Or initialize programmatically at the very top of your entry file:

    <Tabs>
      <Tab title="CommonJS">
        ```javascript server.js
        // MUST be the first line before any other require()
        const tracer = require('dd-trace').init({
          service: 'my-node-app',
          env: 'production',
          logInjection: true
        })

        // Your other imports come after
        const express = require('express')
        ```
      </Tab>
      <Tab title="ESM">
        For ES modules, use the `--import` flag instead:

        ```bash
        node --import dd-trace/initialize.mjs server.mjs
        ```

        Or create a dedicated init file:

        ```javascript register.js
        import 'dd-trace/initialize.mjs'
        ```
        ```bash
        node --import ./register.js server.mjs
        ```
      </Tab>
    </Tabs>

    <Warning>
      Calling `init()` **after** other imports means those libraries will not be instrumented. Always initialize dd-trace first.
    </Warning>
  </Step>

  <Step title="Configure with environment variables">
    You can also configure the tracer with environment variables — no code changes required when using `--require dd-trace/init`:

    ```bash
    DD_SERVICE=my-node-app \
    DD_ENV=production \
    DD_VERSION=1.0.0 \
    DD_LOGS_INJECTION=true \
    node --require dd-trace/init server.js
    ```

    | Environment Variable | Description | Default |
    |---|---|---|
    | `DD_SERVICE` | Service name shown in Datadog | Package name |
    | `DD_ENV` | Environment tag (prod, staging, etc.) | — |
    | `DD_VERSION` | Version tag for your service | — |
    | `DD_AGENT_HOST` | Datadog Agent hostname | `127.0.0.1` |
    | `DD_TRACE_AGENT_PORT` | Datadog Agent port | `8126` |
  </Step>

  <Step title="Run your application">
    Start your application normally. Here's a complete Express example:

    ```javascript server.js
    // MUST be first
    require('dd-trace').init({
      service: 'my-express-app',
      env: 'development'
    })

    const express = require('express')
    const app = express()

    app.get('/', (req, res) => {
      res.json({ message: 'Hello World' })
    })

    app.get('/users/:id', async (req, res) => {
      // dd-trace automatically creates a span for this route
      const user = { id: req.params.id, name: 'Example User' }
      res.json(user)
    })

    app.listen(3000, () => {
      console.log('Server running on port 3000')
    })
    ```

    ```bash
    node server.js
    # OR using --require flag (no code changes needed)
    node --require dd-trace/init server.js
    ```

    Make some requests to your server, then open the [APM Traces page](https://app.datadoghq.com/apm/traces) in Datadog to see your traces.
  </Step>
</Steps>

## Verify traces are being sent

You can enable debug logging to confirm the tracer is working:

```bash
DD_TRACE_DEBUG=true node --require dd-trace/init server.js
```

Look for log output like:
```
Started datadog tracer in development (yourservice)
```

## Configure a specific integration

By default, all built-in plugins are enabled automatically. You can customize how individual integrations behave:

```javascript
const tracer = require('dd-trace').init()

// Customize the PostgreSQL integration
tracer.use('pg', {
  service: 'my-postgres-cluster'
})

// Disable a specific integration
tracer.use('redis', false)
```
