`dd-trace` is the official Datadog APM (Application Performance Monitoring) client library for Node.js. It captures distributed traces from your application and sends them to the Datadog Agent, which forwards them to the Datadog platform for storage, visualization, and alerting.

## How it works

The library sits between your application code and the Node.js module loader. When your app starts, `dd-trace` patches supported libraries automatically — no changes to your business logic required. Each inbound request, outgoing HTTP call, database query, or message queue operation becomes a **span**. Related spans are grouped into a **trace** that represents an end-to-end request through your system.

```
Your Node.js App  →  dd-trace  →  Datadog Agent  →  Datadog Platform
```

The Datadog Agent is a lightweight process that runs alongside your application (on the same host, in a sidecar container, or as a DaemonSet in Kubernetes). It collects spans over HTTP on `localhost:8126` by default, batches them, and forwards them to Datadog.

## Version support

The library follows the Node.js LTS lifecycle. Each major release drops support for Node.js versions that have reached end-of-life.

| Release line | Node.js | SSI | K8s injection | Status | Initial release | End of life |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| `v5` | >= 18 | Yes | Yes | **Current** | 2024-01-11 | — |
| `v4` | >= 16 | Yes | Yes | EOL | 2023-05-12 | 2025-01-11 |
| `v3` | >= 14 | No | Yes | EOL | 2022-08-15 | 2024-05-15 |
| `v2` | >= 12 | No | No | EOL | 2022-01-28 | 2023-08-15 |
| `v1` | >= 12 | No | No | EOL | 2021-07-13 | 2022-02-25 |

**SSI** = Single-Step Install. **EOL** = End of Life.

<Note>
  For all new projects, use `v5`. It is the only actively maintained release line and requires Node.js 18 or later.
</Note>

If you need to run on an EOL version of Node.js, you can install an older release by pinning the major version:

```bash
npm install dd-trace@4
```

EOL release lines no longer receive updates. See the [migration guide](/migration/v4-to-v5) if you are upgrading from an older version.
