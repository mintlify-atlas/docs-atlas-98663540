# Integrations overview

dd-trace ships with built-in plugins for over 100 popular Node.js libraries. When your application loads a supported package, the tracer automatically patches it to capture spans, propagate context, and report metadata to Datadog — no code changes required.

## How the plugin system works

Every supported library corresponds to a named plugin. When you call `tracer.init()`, all built-in plugins are enabled by default. As soon as your code imports a supported module, the plugin intercepts it using Node.js's module loading hooks and diagnostic channels.

Each plugin:
- Creates spans around key operations (HTTP requests, database queries, message publishes, etc.).
- Propagates distributed trace context across service and process boundaries.
- Attaches relevant metadata as span tags (service name, resource, error status, etc.).

## Configure a plugin

Call `tracer.use()` with the plugin name and an options object. Call this **before** importing the library you want to configure.

```javascript
const tracer = require('dd-trace').init()

// Configure the PostgreSQL plugin
tracer.use('pg', {
  service: 'pg-main',
  dbmPropagationMode: 'full'
})

const { Client } = require('pg')
```

You can also configure plugins inside the `init()` call using the `plugins` option:

```javascript
const tracer = require('dd-trace').init({
  plugins: true // enable all built-in plugins (default)
})
```

To disable all built-in plugins and enable only specific ones:

```javascript
const tracer = require('dd-trace').init({
  plugins: false // disable all built-in plugins
})

// Enable only the plugins you need
tracer.use('express')
tracer.use('pg')
```

## Disable a specific plugin

Pass `{ enabled: false }` to `tracer.use()` for the plugin you want to turn off:

```javascript
const tracer = require('dd-trace').init()

// Disable the DNS plugin
tracer.use('dns', { enabled: false })
```

Alternatively, use the environment variable pattern `DD_TRACE_<PLUGIN>_ENABLED=false`. Replace hyphens with underscores and uppercase the plugin name:

```bash
# Disable the fs plugin
DD_TRACE_FS_ENABLED=false node app.js

# Disable the DNS plugin
DD_TRACE_DNS_ENABLED=false node app.js

# Disable the kafkajs plugin
DD_TRACE_KAFKAJS_ENABLED=false node app.js
```

<Warning>
Disabling plugins can cause unexpected side effects such as broken context propagation. Leave plugins enabled unless you have a specific reason to disable them.
</Warning>

## Use span hooks

Web framework plugins expose a `hooks.request` callback that runs just before the request span finishes. Use it to attach custom tags without writing a custom instrumentation layer:

```javascript
const tracer = require('dd-trace').init()

tracer.use('express', {
  hooks: {
    request: (span, req, res) => {
      span.setTag('customer.id', req.query.customerId)
      span.setTag('tenant', req.headers['x-tenant-id'])
    }
  }
})
```
