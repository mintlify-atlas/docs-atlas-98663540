---
title: 'Environment variables'
description: 'Complete reference for all dd-trace environment variables grouped by feature area.'
---

All environment variables recognized by `dd-trace`. Programmatic options set in `tracer.init()` override the corresponding environment variable unless noted otherwise.

<Note>
  `DD_LLMOBS_ENABLED` is the exception: the environment variable takes precedence over the programmatic `llmobs.enabled` option.
</Note>

## Core

| Variable | Default | Description |
|---|---|---|
| `DD_SERVICE` | *(inferred from package.json)* | Service name. Also accepts alias `DD_SERVICE_NAME`. |
| `DD_ENV` | — | Deployment environment (e.g. `prod`, `staging`). |
| `DD_VERSION` | *(inferred from package.json)* | Application version. |
| `DD_AGENT_HOST` | `127.0.0.1` | Hostname of the Datadog Agent. Also accepts alias `DD_TRACE_AGENT_HOSTNAME`. |
| `DD_TRACE_AGENT_PORT` | `8126` | Port of the Datadog Agent. |
| `DD_TRACE_AGENT_URL` | — | Full URL of the Datadog Agent (overrides `DD_AGENT_HOST` + `DD_TRACE_AGENT_PORT`). Also accepts alias `DD_TRACE_URL`. |
| `DD_SITE` | `datadoghq.com` | Datadog intake site for agentless features. |
| `DD_API_KEY` | — | Datadog API key (required for agentless features). Also accepts alias `DATADOG_API_KEY`. |
| `DD_TAGS` | — | Comma-separated global tags applied to every span, e.g. `team:backend,region:us-east`. |
| `DD_SERVICE_MAPPING` | — | Comma-separated per-plugin service name overrides, e.g. `pg:postgres-primary,redis:cache`. |
| `DD_APM_TRACING_ENABLED` | `true` | Set to `false` to disable APM tracing while keeping standalone products active. |
| `DD_TRACE_ENABLED` | `true` | Master switch for the tracer. |
| `DD_TRACE_DEBUG` | `false` | Enable debug logging. |
| `DD_TRACE_LOG_LEVEL` | `debug` | Minimum log level when debug is on. One of `debug`, `info`, `warn`, `error`. Also configurable via `OTEL_LOG_LEVEL`. |
| `DD_TRACE_STARTUP_LOGS` | `false` | Print startup diagnostics to the console. |
| `DD_TRACE_REPORT_HOSTNAME` | `false` | Include the service host's hostname as a span tag. |
| `DD_TRACE_AGENT_PROTOCOL_VERSION` | `0.4` | Agent protocol version. |
| `DD_LOGS_INJECTION` | `true` | Inject trace and span IDs into log records. |
| `DD_LOG_LEVEL` | — | Override the tracer internal log level. |

## Tracing and sampling

| Variable | Default | Description |
|---|---|---|
| `DD_TRACE_SAMPLE_RATE` | — | Global ingestion sample rate (0–1). Also configurable via `OTEL_TRACES_SAMPLER` / `OTEL_TRACES_SAMPLER_ARG`. |
| `DD_TRACE_RATE_LIMIT` | *(agent decision)* | Global trace rate limit (traces per second). |
| `DD_TRACE_SAMPLING_RULES` | `[]` | JSON array of sampling rules: `[{"service":"my-api","sampleRate":1.0}]`. |
| `DD_SPAN_SAMPLING_RULES` | — | JSON array of span-level sampling rules for keeping individual spans from dropped traces. |
| `DD_SPAN_SAMPLING_RULES_FILE` | — | Path to a JSON file containing span sampling rules. |
| `DD_TRACE_FLUSH_INTERVAL` | `2000` | Trace flush interval in milliseconds. |
| `DD_TRACE_PARTIAL_FLUSH_MIN_SPANS` | `1000` | Number of spans that triggers a partial export. |
| `DD_TRACE_128_BIT_TRACEID_GENERATION_ENABLED` | `true` | Generate 128-bit trace IDs. |
| `DD_TRACE_128_BIT_TRACEID_LOGGING_ENABLED` | `true` | Log trace IDs as 128-bit values. |

## Propagation

| Variable | Default | Description |
|---|---|---|
| `DD_TRACE_PROPAGATION_STYLE` | — | Comma-separated list of propagation formats for both inject and extract (e.g. `datadog,tracecontext`). |
| `DD_TRACE_PROPAGATION_STYLE_INJECT` | — | Propagation formats used for injection only. |
| `DD_TRACE_PROPAGATION_STYLE_EXTRACT` | — | Propagation formats used for extraction only (highest priority listed first). |
| `OTEL_PROPAGATORS` | — | OpenTelemetry propagator names. Used when DD-specific propagation vars are not set. |
| `DD_TRACE_BAGGAGE_MAX_BYTES` | `8192` | Maximum size in bytes for baggage propagation. |
| `DD_TRACE_BAGGAGE_MAX_ITEMS` | `64` | Maximum number of baggage items. |
| `DD_TRACE_BAGGAGE_TAG_KEYS` | `user.id, session.id, account.id` | Comma-separated baggage keys automatically promoted to span tags. |

| `DD_APPSEC_OBFUSCATION_PARAMETER_VALUE_REGEXP` | *(built-in pattern)* | Regex to redact sensitive parameter values in attack reports. |
| `DD_APPSEC_HTTP_BLOCKED_TEMPLATE_HTML` | — | Path to custom HTML block page template. |
| `DD_APPSEC_HTTP_BLOCKED_TEMPLATE_JSON` | — | Path to custom JSON block response template. |

| `DD_APPSEC_RASP_ENABLED` | `true` | Enable Runtime Application Self-Protection. |
| `DD_APPSEC_STACK_TRACE_ENABLED` | `true` | Include stack traces in attack reports. |
| `DD_APPSEC_MAX_STACK_TRACES` | `2` | Maximum number of stack traces per attack report. |
| `DD_APPSEC_MAX_STACK_TRACE_DEPTH` | `32` | Maximum depth of each stack trace. |
| `DD_API_SECURITY_ENABLED` | `true` | Enable API Security schema collection. |
| `DD_API_SECURITY_ENDPOINT_COLLECTION_ENABLED` | `true` | Enable endpoint collection for API Security. |
| `DD_API_SECURITY_ENDPOINT_COLLECTION_MESSAGE_LIMIT` | `300` | Maximum endpoints serialized per message. |

| `DD_IAST_SECURITY_CONTROLS_CONFIGURATION` | — | Security controls configuration string. |
